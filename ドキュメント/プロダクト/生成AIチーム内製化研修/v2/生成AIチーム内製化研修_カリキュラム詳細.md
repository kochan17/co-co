# 生成AIチーム内製化研修 カリキュラム詳細
パートナー連携・内製化支援：v2

## 1. 研修の狙い
「社内にAI開発・推進機能を構築し、外部依存からの脱却と企業の独自進化を実現する（生成AIネイティブ人材の育成とプロの武器の提供）」

## 2. v1との違い
本バージョン（v2）では、従来の「個人のAI活用スキルの向上」という枠組みを超え、**「組織資産としてのAI開発」**と**「事例共有体制の構築」**を重視しています。

**■「使い方」から「作り方」へ**

  GPTs構築等のGUI操作に留まらず、Python/VBAを用いた本格的な業務自動化をカリキュラムの核に据えました。

**■「個人」から「チーム」へ**

  AI知見が個人に閉じ、社内に浸透しない「属人化パズル」を解消するため、AI推進チームによるコミュニケーション体制의 構築を必須プロセスとして組み込みました。

## 3. 他社研修・一般研修との決定的な違い
本プログラムは、単なる「AIの使い方」を教える研修ではありません。以下の3点において市場の一般的研修と差別化されています。

| 比較項目 | 一般的なAI研修 | 本プログラム (v2) |
| :--- | :--- | :--- |
| **技術的深さ** | GUIツール（GPTs構築等）の範疇 | **Python / VBA等を用いた「本格開発・自動化」** |
| **解決の広さ** | 個人の効率化（点） | **複雑な業務プロセスの根本改善（面）** |
| **浸透の仕組み** | 個人任せ（ブラックボックス化） | **「AI推進チーム」による事例共有・組織化** |

### 独自の強み：インパクトの最大化と属人化の打破

**■開発への踏み込み**

  ChatGPTやGeminiのGUIだけでは届かない「複雑な定型業務の自動化」を、PythonやVBAを駆使して実現します。

**■「属人化パズル」を解く組織化**

  生成AIは個人の指示（プロンプト）に依存するため、活用がブラックボックス化しがちです。

  本研修では、詳しい人を「チーム」化し、事例や情報を共有し合えるコミュニケーション体制を構築することで、組織全体への浸透を確実なものにします。

## 3. カリキュラム構成
※枠組みは共通ですが、各ステップで具体的な「実戦アセット」の構築を目指します。

## STEP 1: 開発基盤の構築
AIを単なるチャットツールではなく、開発パートナーとして活用するための土台を作ります。

**■Git/GitHub基礎**

  バージョン管理による「社内知見の資産化」。

**■IDE / Agentic AI環境**

  Claude Code / Antigravityの設定と実務への即時導入。

**■AIガバナンス研修**

  セキュリティを担保した上でAIを使い倒すためのルール策定。

## STEP 2: 実践・内製化
実務課題をAIと共に解決し、独自のツール（業務アセット）を構築する力を養います。

**■AIプログラミング実践**

  PythonやVBAを活用し、現場担当者が自ら業務を自動化。

**■チーム開発の作法**

  プルリクエストベースの開発フローによる「脱・属人化」。

**■ワークフロー自動化**

  DifyやPowerAutomateを活用した業務プロセスの内製化。

## STEP 3: 組織トランスフォーメーション
AI活用を組織全体に広げ、継続的な改善サイクル（自走体制）を構築します。

**■社内AI推進チームの発足**

  研修修了生を中心とした、部門横断的な推進組織の立ち上げ。

**■コミュニケーション基盤の構築**

  実装事例の共有会や、プロンプト集の活用による組織リテラシー向上。

**■AI顧問サービス**

  最新テクノロジーのキャッチアップと継続的な技術相談。

## 4. 目標とする到達像

**■無駄なシステム費用の削減**

  不要な外部SaaSの代替による固定費の最適化。

**■外注依存からの脱却**

  軽微な改修やプロトタイプ開発の完全内製化。

**■非エンジニアの開発力習得**

  現場担当者が自律的にデジタルツールを構築・改善。

**■社内AI推進チームの発足**

  知見共有の仕組みを構築し、全社へAI活用を浸透させる。
