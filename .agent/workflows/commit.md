---
description: コミット・プッシュ ワークフロー
---

現在の変更点を確認し、まとまりのある単位でコミットしてリモートリポジトリにプッシュします。

## 手順

1. `git status` で変更ファイルを確認する
2. 関連する変更をまとめて `git add` する
3. **日本語**でコミットメッセージを作成する
   - 例: `ArticleWritingSkillの構成を整理：リソースとスクリプトをスキルフォルダ内に集約`
4. `git push origin main` でリモートにプッシュする
5. 複数のまとまりがある場合は、それぞれ別のコミットとして分割する

## コミットメッセージのルール
- 日本語で記述すること
- 何を変更したかが一目でわかる内容にすること
- 必要に応じて「：」で区切って詳細を追記する
