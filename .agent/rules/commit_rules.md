# コミットおよびプッシュのルール

Gitでの変更保存（コミット）およびリモートへの反映（プッシュ）を行う際は、以下のルールを厳守してください。

## コミットメッセージの規約
- **言語**: 必ず**日本語**で記述すること。
- **内容**: 変更内容が具体的かつ一目でわかるようにすること。
- **形式**: 必要に応じて「：」で区切って詳細を追記すること。
  - 例: `ArticleWritingSkillの構成を整理：リソースとスクリプトをスキルフォルダ内に集約`

## 手順の基本方針
1. `git status` で変更内容を常に把握する。
2. 関連する変更は論理的な単位でまとめ、適切に `git add` する。
3. 複数の独立した変更がある場合は、コミットを分割すること。
4. 原則として、コミット後は `git push origin main`（または適切なブランチ）でリモートに反映すること。

> [!NOTE]
> 明示的な手順実行が必要な場合は、`/commit` ワークフローを使用することも可能です。
