# プロジェクト命名・言語・アーティファクト規定

このプロジェクトでは、円滑なコミュニケーションとドキュメントの可読性を維持するため、以下の命名および言語ルールを厳守してください。

## 1. アーティファクト（`.gemini/antigravity/brain/` 配下）
**すべてのMarkdownファイル（`.md`）の内容およびタイトルは日本語で記述してください。**
- `task.md`: タスクリスト、進捗管理
- `implementation_plan.md`: 実施計画、設計、リスク確認
- `walkthrough.md`: 成果報告、検証結果
- その他生成されるすべてのドキュメント

## 2. ファイル名・フォルダ名
**新規作成・既存改名を問わず、すべてのファイル名とフォルダ名は日本語で命名してください。**
- 例: `docs/` → `ドキュメント/`
- 例: `README.md` → `README_プロジェクト概要.md`
- **注意**: システム的に必須な隠しファイル（`.git`, `.agent` 等）や設定ファイル（`CLAUDE.md`）は除きます。

## 3. ドキュメントの内容
プロジェクトに関連するドキュメント（`docs/` 配下など）の内容も、原則として日本語で記述してください。

---
このルールは、ユーザーとの迅速な合意形成と、プロジェクトの長期的なメンテナンス性を確保するために設定されています。
