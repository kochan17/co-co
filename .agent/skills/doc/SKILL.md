# doc スキル

このスキルは、co-coプロジェクトにおける全てのMarkdownドキュメントの品質と一貫性を担保するためのものです。戦略コンサルタントレベルの品格と、デジタル環境での読みやすさを両立させる「co-co標準スタイル」を適用します。

## 核心的な基本原則

1. **視覚的なスキャン性の最大化**: 重要な項目を一目で捉えられるようにする。
2. **プレビュー品質の保証**: 各種Markdownプレビュー環境で文が詰まらず、美しく表示されるようにする。
3. **論理的構造**: 階層構造（Heading）をクリーンに保ち、情報設計を整理する。

## co-co標準スタイル規約

### 1. 見出し（Headings）
- カッコ`()`は使用禁止。
- バージョンや補足事項は見出し内に入れず、見出し直下に記述。
- 例：`## 3. カリキュラム構成` （「(3ステップ)」等は不要）

### 2. 項目タイトル（■形式）
- 強調したい項目やラベルは **`**■タイトル**`** 形式で独立した行に出力する。
- 箇条書き（`- **タイトル**:`）や単純な太字（`**タイトル**`）は原則として使用しない。

### 3. 本文の構造
- **インデント**: タイトルの下に続く内容は、半角スペース2つ分インデントする。
- **改行**: 文章（。や？）ごとに**必ず空行（ダブルリターン）**を挿入する。これにより、プレビュー時に段落が適切に分離され、一行ずつの表示が維持される。

### 4. 戦略的文体
- 「〜です・ます」調を基本としつつ、ロジックに基づいたプロフェッショナルなトーン。
- ROI、資産化、内製化、属人化といったキーワードを適切に用い、経営的価値を明確にする。

## 使用手順

1. **初期分析**: 元となる情報やユーザーの要求を分析し、最適な構成案を作成する。
2. **ドラフト作成**: 上記規約を全て適用して執筆する。
3. **セルフレビュー**: 
   - 全ての強調タイトルが `■` 形式になっているか？
   - 文章の間に空行があるか？
   - 変なカッコ付き見出しがないか？
4. **アウトプット**: ユーザーに提示し、必要に応じて微調整を行う。
